# Abstract

This proposal introduces a two-part initiative by Kann Audits to strengthen the security posture of the Canton ecosystem.

First, a Security Partnership Program providing ongoing security advisory, prioritized audit support, and incident response assistance for builders across the ecosystem.

Second, a Development Fund request for DAML-native AI security tooling designed to enable automated vulnerability detection, continuous monitoring, and improved security analysis of Canton smart contracts and applications.

Together, these initiatives establish a durable ecosystem security layer that reduces developer friction, improves protocol safety, and supports the secure scaling of applications built on Canton.

As adoption of Canton increases, builders face growing demand for specialized security expertise and scalable review infrastructure. Traditional audits alone are insufficient to support continuous development cycles. This proposal addresses this gap by combining human-led security expertise with AI-assisted analysis and continuous monitoring.

The combined initiative focuses on:
- Continuous security advisory and hands-on support for Canton builders  
- Prioritized and discounted manual audit services for Canton-native projects  
- Incident response readiness for critical vulnerabilities and exploits  
- DAML-native AI security tooling for automated vulnerability detection  
- Continuous monitoring and security lifecycle coverage for deployed protocols  

!!!
The Security Partnership Program (including manual audits, advisory support, and incident response) is provided independently of the Development Fund request and is available immediately to Canton builders.

The Development Fund request applies solely to the DAML-native AI security tooling and monitoring, which will be delivered through milestone-based development and ecosystem rollout.

# 1. Objective

Kann Audits proposes establishing a shared ecosystem security infrastructure layer for Canton through a combination of human expertise, manual audit capability, incident response, and AI-powered tooling.

Specifically, Kann Audits commits to:
- Build and maintain DAML-native AI security tooling tailored for Canton smart contracts and applications  
- Provide continuous security advisory support for ecosystem builders  
- Deliver discounted and prioritized manual and AI-assisted audit services  
- Maintain incident response readiness for ecosystem security events and critical vulnerabilities  
- Provide continuous monitoring and security review capabilities for active protocols  
- Support ecosystem adoption through security workshops and developer enablement

# 2. Implementation Mechanics

The proposal introduces a two-layer ecosystem security model:

## Layer 1 — Ecosystem Security Partnership

Kann Audits will act as a long-term security partner for Canton builders by providing:

- Security architecture consultations during development  
- Smart contract and protocol design reviews  
- On-demand security guidance for teams building on Canton  
- Secure development best practices and guidance  

### Manual Audit Incentive Program:
- 30% discount on first audit engagement for Canton-native projects  
- 10% discount on future audit engagements  
- Priority audit scheduling for Canton ecosystem builders  
- 12 months of post-audit remediation and consultation support  

### Incident Response Coverage:
- Rapid vulnerability triage  
- Exploit mitigation guidance  
- Coordination support during critical security events  

---

## Layer 2 — DAML AI Security Audit Infrastructure

Kann Audits will develop a DAML-native AI security analysis system tailored specifically for Canton architecture.

The tooling will include:
- Static analysis for DAML smart contracts  
- Semantic vulnerability detection using AI reasoning models  
- Pattern recognition for authorization and contract design flaws  
- Cross-contract dependency and state-flow analysis  
- Automated security report generation (human-readable outputs)  
- Continuous monitoring for deployed protocols  

### Operational AI Coverage Model:
- Ecosystem allocation: up to 7 protocols per month covered under continuous AI monitoring  
- Each allocated protocol includes AI-assisted scanning with human validation  
- Structured security reports including severity classification and remediation guidance  

For protocols outside allocated coverage, additional AI-assisted scans and extended monitoring will be available as a discounted service to ensure sustainable operation of the program.

- Improved cost efficiency by reducing re-audit cycles, bug bounty exposure, and post-deployment fixes

# 10. Why This Matters

This proposal establishes a continuous security layer for the Canton ecosystem that integrates security directly into the development lifecycle.

It moves the ecosystem from isolated, point-in-time audits toward a hybrid model combining AI-assisted analysis, expert manual review, and incident response capability.

The result is a more resilient builder environment where security is continuous, scalable, and embedded throughout development rather than treated as a final step.

# 11. Motivation

This proposal is motivated by the lack of an end-to-end security support layer in the Canton ecosystem that covers the full lifecycle of application development.

Today, security support is fragmented across early-stage advisory, manual audits, and post-deployment monitoring.

We aim to close this gap by introducing a unified security pathway that supports builders from initial design through development, audit, deployment, and ongoing operational security.
